Software Requirements Specification (SRS)

No.: Requirement: Description: regulatory functional
1 Hierarchical structure Requirements are stored in a tree structure of arbitrary depth. The following editing actions are possible adding and removing requirements moving requirements copying requirements
2 The structure is editable The requirement tree structure can be edited by the user.
3 Requirement types One of three different "types" can be assigned to each requirement. The type of each requirement is made visible in the requirement tree view. This provides the possibility to structure the requirements tree into three layers of depth: User Requirements Functional Requirements Design Requirements
4 Rich text descriptions For each requirement a rich text description can be edited to specify its details.
5 Saving and Loading of Projects All information edited in the software can be saved to file and loaded again.
6 Exporting Reports The information stored in a project file can be exported to reports. Possible formats are HTML PDF OpenDocument
7 Digital Signatures Project files and/or exported reports can be digitally signed and signatures can be veryfied using the Requirements Manager or an additional tool.
8 Attributes For each project an arbitrary number of user-configurable "requirement attributes" can be used. Attributes store additional information about requirements e.g. their state, a time estimation, a comment ... Attributes have the following properties name data type value boolean text ranking (1 to 5) integer
9 Links For each project an arbitrary number of user-configurable "requirement links" can be used. Links can be used to specify relationships like "requirement A affects / contradicts / ... requirements B, C, D, ..."
10 Risk Assessment For each requirement an assessment of their risks can be edited: Several "risk scenarios" can be added to each requirement Several "preventive actions" can be assigned to each risk scenario Each risk risk scenario has a final risk resulting from the sum of preventive actions
11 Source Code Extraction When used with software projects the Requirements Manager can extract declarations of classes, functions and automated tests along with their comments from the project's source files. Classes and functions can be added as Source Code Documentation to the Design Requirements Automated tests can be added to risk assessments
12 Full-Text Search The user can perform a full text search in the requirements docuement.
